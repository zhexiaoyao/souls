# 游戏独立项目说明

## 概述

这个目录包含了3D收集游戏的独立版本，可以单独编译和运行，不依赖主引擎项目的构建系统。

## 文件结构

```
game/
├── CMakeLists.txt          # 独立的CMake构建配置
├── main.cpp                # 游戏主程序入口（独立版本）
├── README.md               # 项目说明
└── 游戏项目说明.md         # 本文件
```

## 编译方式

### 方式1：在主项目中编译（推荐）

在主项目 `Souls-Engine/build` 目录下：
```bash
cmake --build . --config Debug --target SoulsEngine_Game
```

### 方式2：独立编译

1. 进入游戏目录：
```bash
cd Souls-Engine/game
mkdir build
cd build
```

2. 配置CMake：
```bash
cmake ..
```

3. 编译：
```bash
cmake --build . --config Debug
```

4. 运行：
```bash
cd bin\Debug
CollectGame.exe
```

## 代码封装说明

### 游戏核心组件

1. **GameManager** (`src/core/GameManager.h/cpp`)
   - 游戏逻辑管理
   - 玩家控制
   - 碰撞检测
   - 游戏状态管理

2. **游戏主程序** (`game/main.cpp`)
   - 独立的游戏入口点
   - 使用相对路径引用引擎核心代码
   - 完整的游戏循环和UI渲染

### 依赖关系

游戏依赖以下引擎组件（通过相对路径引用）：
- `../src/core/` - 核心系统
- `../src/geometry/` - 几何体
- `../extern/imgui/` - UI库
- `../assets/` - 资源文件

## 优势

1. **独立性**: 可以作为独立项目分发
2. **封装性**: 游戏逻辑与引擎代码分离
3. **可维护性**: 游戏代码集中在一个目录
4. **可扩展性**: 易于添加新游戏功能

## 注意事项

- 游戏需要访问父目录的引擎代码和资源
- 编译时需要确保所有依赖路径正确
- Shader文件路径会自动搜索多个位置
